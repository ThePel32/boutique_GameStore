Projet : Application Web Gamestore

1. Introduction

L'objectif de ce document est de définir les spécifications fonctionnelles de 
l'application web Gamestore. Cette application vise à moderniser l'approche commerciale 
de l'entreprise en permettant aux clients de réserver des jeux en ligne et de les 
récupérer en magasin. Le cahier des charges servira de guide pour le développement 
de l'application en définissant clairement les fonctionnalités attendues et les 
contraintes techniques à respecter.

2. Description du Projet

Gamestore souhaite développer une application web et web mobile permettant aux 
clients de consulter les jeux disponibles, de les réserver en ligne et de les récupérer
 dans le magasin de leur choix. L'application comprendra deux interfaces 
 distinctes : une partie visiteur accessible à tous les utilisateurs et une partie 
 administrateur réservée au personnel de l'entreprise.

3. Fonctionnalités de la Partie Visiteur

    * Page d'accueil : Présenter l'entreprise, les promotions en cours et les jeux récents.
    * Menu de navigation : Faciliter l'accès aux différentes sections de l'application.
    * Vue globale des jeux vidéo : Afficher tous les jeux disponibles avec possibilité de filtrage par prix et genre.
    * Vue détaillée d'un jeu vidéo : Afficher toutes les informations détaillées sur un jeu sélectionné.
    * Ajout au panier : Permettre aux utilisateurs d'ajouter des jeux au panier après s'être authentifiés.
    * Panier d'achat : Gérer les articles ajoutés au panier, y compris la sélection du lieu et de la date de retrait.
    * Espace utilisateur : Permettre aux utilisateurs de consulter leur historique de commandes et de modifier 
    leurs informations personnelles.
    * Création de compte utilisateur : Permettre aux visiteurs de créer un compte utilisateur avec confirmation par e-mail.
    * Réinitialisation de mot de passe : Permettre aux utilisateurs de réinitialiser leur mot de passe en cas d'oubli.

4. Fonctionnalités de la Partie Administrateur

    * Gestion des jeux vidéo : Ajouter, modifier et supprimer des jeux vidéo avec leurs informations associées.
    * Gestion des stocks : Ajouter et mettre à jour les quantités disponibles pour chaque jeu vidéo.
    * Dashboard des ventes : Afficher les statistiques de vente par genre et par date.
    * Gestion des utilisateurs : Ajouter et supprimer des comptes employés.

5. Contraintes Techniques

    * L'application doit être développée en utilisant HTML, CSS et JavaScript pour le front-end, Node.js avec 
    Express pour le back-end, et MySQL pour la base de données.
    * Les mots de passe des utilisateurs doivent être hachés avant d'être stockés dans la base de données 
    pour des raisons de sécurité.
    * L'application doit être déployée sur Heroku pour permettre un accès en ligne.
